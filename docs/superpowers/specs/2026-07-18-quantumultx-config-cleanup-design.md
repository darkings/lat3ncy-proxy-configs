# Quantumult X 配置清理与分流设计

## 目标

在不改变知乎现有净化规则的前提下，降低配置泄密风险、减少无用后台任务，并明确区分中国版抖音与国际版 TikTok 的分流。

## 配置变更

- 从主配置彻底删除节点订阅 URL、MITM `passphrase` 和 `p12`，不在仓库保留副本；保留空白说明，节点订阅由用户在设备端重新添加，MITM 证书由 Quantumult X 重新生成。
- 删除“美团 Cookie 自动获取”远程重写。
- 注释汇率监控、钉钉打卡、每日一言和今日黄历；保留今日油价、节假提醒、Speed 测速和节点位置查询。
- DNS 精简为 `223.5.5.5` 与 `119.29.29.29`，保留现有 iCloud、腾讯和微信域名定向解析。
- 新增“国内服务”静态策略组，以 `direct` 为默认选项，并保留 `proxy` 作为手动排障选项。
- 将微信、国内域名和国内 ASN 规则的强制策略改为“国内服务”。
- 在本地分流中加入抖音常用域名后缀直连规则，并指向“国内服务”。
- 保留国际版 TikTok 的独立策略组、远程分流和 `_region=CN` 到 JP 的地区重写。
- 保留 fmz200 的轻量抖音规则，将标签改为“抖音轻量净化@fmz200”，不再称为 TikTok 国际版净化。
- 知乎远程重写保持不变。

## 注释清理范围

删除与当前启用配置无关的教程、协议示例、长期禁用的订阅示例、空功能标题和重复说明。保留章节说明、关键风险说明、被本次注释且可能恢复的四类任务，以及当前有效规则的用途注释。

## GitHub 脚本评估原则

本次不直接新增第三方脚本。研究结果按以下标准给出候选清单：仓库仍维护、与手机现有 App 匹配、没有与现有规则明显重复、不修改会员权益或账户凭据、收益高于 MITM 和误拦截风险。优先考虑手动交互查询或轻量分流，谨慎考虑复杂响应体改写和签到脚本。

## 验证

由于仓库只有配置文件且没有测试框架，本次不建立代码单元测试。修改后执行配置结构与关键断言检查：章节唯一且顺序正确；敏感字段和订阅 token 不存在；策略引用均有定义；四类任务已禁用而油价、节假提醒和两个交互任务仍启用；抖音与 TikTok 分流同时存在；远程资源 URL 格式可解析。
